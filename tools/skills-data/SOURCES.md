# PalBreed Skills data sources

The browser-ready dataset at `assets/data/skills.js` is generated by
`generate.mjs`. The generator verifies every pinned input by SHA-256 before it
writes output.

## Primary source

- Project: [tylercamp/palcalc](https://github.com/tylercamp/palcalc)
- Snapshot: PalCalc `v26`, commit `8ade7381d0548d0312dbe0d650673e4cebd86fd6`
- File: `PalCalc.Model/db.json`
- License: MIT (as declared by the GitHub repository API)
- Used for: the canonical active/passive roster, English names and passive
  descriptions, elements, power, cooldown, inheritance flags, Skill Fruit
  availability, and guaranteed Pal passives.

## Secondary source

- Project: [oMaN-Rod/palworld-save-pal](https://github.com/oMaN-Rod/palworld-save-pal)
- Snapshot: commit `4cd0a85f66cf424b650ef0e886032a6b7419e55b`
- Files: `data/json/pals.json`, `active_skills.json`, and
  `passive_skills.json`
- License: `NOASSERTION`. GitHub's repository API did not declare a license at
  generation time.
- Used for: Pal level-up learnsets and supplemental active range/type/effect and
  passive effect fields. Joins use exact internal identifiers; unmatched fields
  remain null or empty.

## Scope decisions

- The passive array contains the 115 records that PalCalc marks as standard
  passive skills. Internal equipment, test, and non-Pal modifier records are
  intentionally excluded from the user-facing dataset.
- `active[].exclusive` is the inverse of PalCalc's `CanInherit` field.
- `passive[].exclusive` is the inverse of PalCalc's
  `RandomInheritanceAllowed` field.
- Neither pinned source exposes a normalized Partner Skill table, so `partner`
  is an intentionally empty array and the limitation is recorded in metadata.
- Active-skill descriptions are not provided by either pinned source and remain
  `null`; no descriptions are inferred or fabricated.
- Two secondary learnset entries reference Tetroise skill identifiers absent
  from PalCalc's active roster. They are preserved in
  `meta.unresolvedLearnsets`, while the main active array remains a faithful
  PalCalc roster.

## Regeneration

From the repository root:

```text
node tools/skills-data/generate.mjs
node tools/skills-data/validate.mjs
```
